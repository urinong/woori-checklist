## Part 2: 배포 준비

### STEP 1: 배포용 파일 생성

프로젝트 루트에 아래 파일들을 생성하세요.

#### Procfile (확장자 없음, 프로젝트 루트)
```
web: gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120
```

#### runtime.txt (프로젝트 루트)
```
python-3.11.11
```

#### requirements.txt에 gunicorn 추가
기존 requirements.txt에 `gunicorn`이 없으면 추가:
```
gunicorn
```

### STEP 2: app.py 포트 설정 확인

app.py의 마지막 실행 부분이 다음과 같은지 확인하고, 아니면 수정:

```python
if __name__ == '__main__':
    port = int(os.environ.get('PORT', 5000))
    app.run(host='0.0.0.0', port=port, debug=False)
```

- `debug=True`이면 `debug=False`로 변경 (프로덕션)
- `host='0.0.0.0'`이어야 함 (외부 접속 허용)
- `PORT` 환경변수를 읽어야 함 (Render가 동적으로 포트 할당)

### STEP 3: uploads/outputs 폴더 처리

Render는 재시작 시 파일이 삭제되므로, 앱 시작 시 폴더를 자동 생성하도록 확인:

```python
# app.py 상단 또는 config.py에
import os
os.makedirs('uploads', exist_ok=True)
os.makedirs('outputs', exist_ok=True)
```

### STEP 4: .gitignore 확인

프로젝트 루트에 .gitignore가 없으면 생성:

```
__pycache__/
*.pyc
.env
uploads/*
outputs/*
!uploads/.gitkeep
!outputs/.gitkeep
venv/
.venv/
```

uploads/와 outputs/ 폴더에 .gitkeep 파일 생성:
```bash
touch uploads/.gitkeep outputs/.gitkeep
```

### STEP 5: README.md 업데이트

```markdown
# 우리농 신규물품 출하 준비 자가진단 체크리스트

교구 실무자를 위한 신규물품 출하 서류 점검 도구

- **서류 검증**: 받은 서류가 완비되었는지, 내용이 맞는지 점검합니다

## 로컬 실행
```bash

## 환경변수 (선택)

## Part 3: GitHub 푸시 및 Render 배포

### Claude Code에서 실행할 Git 명령어

```bash
# 1. Git 초기화 (이미 되어있으면 스킵)

# 2. 전체 파일 추가

# 3. 커밋
git commit -m "우리농 자가진단 체크리스트 v1.0 - 배포 준비 완료"

# 4. GitHub 원격 저장소 연결
git remote add origin https://github.com/urinong/woori-checklist.git

# 5. 푸시
git branch -M main
git push -u origin main
```

⚠️ GitHub에 `woori-checklist` 저장소가 없으면 먼저 만들어야 합니다.
토니가 https://github.com/new 에서 생성하거나, GitHub CLI로:
```bash
gh repo create urinong/woori-checklist --public --source=. --remote=origin --push
```

### Render 배포 절차 (토니가 브라우저에서 수행)

1. https://render.com 접속 → GitHub 계정으로 가입/로그인
2. Dashboard → **New +** → **Web Service**
3. GitHub 저장소 연결: `urinong/woori-checklist` 선택
4. 설정:
   - **Name**: `woori-checklist` (URL이 woori-checklist.onrender.com이 됨)
   - **Region**: Singapore (한국에서 가장 가까운 리전)
   - **Branch**: main
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120`
   - **Plan**: Free
5. **Environment Variables** (환경변수):
   - Key: `ANTHROPIC_API_KEY` / Value: `sk-ant-...` (토니의 API 키)
   - ※ API 키가 없으면 이 단계 스킵 가능 (포장지 OCR만 비활성화)
6. **Create Web Service** 클릭
7. 빌드 완료 후 (약 2-5분) URL 확인: `https://woori-checklist.onrender.com`

### 배포 후 확인

1. URL 접속하여 화면 정상 표시 확인
2. 기능 A: 바나나송송 물품사양서 PDF 업로드 → 체크리스트 생성 확인
3. 기능 B: 가자미구이 합본 PDF 업로드 → 서류 검증 결과 확인

### 첫 접속 시 주의
Render 무료 티어는 15분 미사용 시 서버가 슬립 상태가 됩니다.
첫 접속 시 30초~1분 정도 로딩이 걸릴 수 있어요.
"잠시만 기다려주세요" 안내가 있으면 좋습니다.

### 접속 비밀번호 (선택)
내부 문서(물품사양서 등)를 다루므로, 간단한 접속 비밀번호를 설정하고 싶다면:

Render 환경변수에 추가:
- Key: `ACCESS_PASSWORD` / Value: `원하는비밀번호`

app.py에 미들웨어 추가:
```python
ACCESS_PASSWORD = os.environ.get('ACCESS_PASSWORD')

@app.before_request
def check_password():
    if ACCESS_PASSWORD and request.endpoint != 'static':
        if not session.get('authenticated'):
            if request.method == 'POST' and request.form.get('password') == ACCESS_PASSWORD:
                session['authenticated'] = True
                return redirect('/')
            return render_template('login.html')
```

이 부분은 필요하면 나중에 추가해도 됩니다.
