# 우리농 신규물품 출하 준비 자가진단 체크리스트 — Claude Code 작업지시서 (v2)

> ⚠ 이 문서는 기존 작업지시서(05_ClaudeCode_작업지시서.md)를 **대체**합니다.
> 기존의 "심의 보조 시스템" 설계를 폐기하고, 교구 실무자용 자가진단 도구로 재설계합니다.

## 프로젝트 개요

### 무엇을 만드는가
**우리농 신규물품 출하 준비 자가진단 체크리스트** — 지역 교구 우리농 실무자가 생산자/협력업체로부터 신규 물품 출하 서류를 요청하거나 받았을 때, 우리농 생산규정에 맞는지 스스로 점검할 수 있는 웹앱.

### 누가 사용하는가
- 지역 교구 우리농 생산소비 담당 실무자
- 신입직원도 체크리스트만 보면 이해할 수 있어야 함
- IT 지식 없이 브라우저에서 바로 사용

### 핵심 원칙
1. **신입직원 기준**: 모든 체크 항목에 "이게 왜 필요한지" 설명이 달려 있어야 함
2. **자가진단**: 실무자가 스스로 빠진 게 없는지 확인하는 도구. 심의 판정을 내리는 게 아님
3. **사전 대비**: 출하 승인 과정에서 이슈될 수 있는 부분을 미리 알려줘서 대비할 수 있게

## 두 가지 기능

### 기능 A: 빠른 진단 — "이 물품, 출하 준비 뭐가 필요해?"

**사용 시나리오**: 생산자가 "이 제품 우리농에 출하하고 싶다"고 연락했을 때, 실무자가 제품 포장지 사진이나 물품사양서만 갖고 빠르게 "어떤 서류를 요청해야 하는지" 파악

**입력** (둘 중 하나):
- 가공식품 포장지 표시사항 사진 (JPG/PNG) — 핸드폰으로 찍은 것
- 물품사양서 PDF

**출력**: 체크리스트 형태의 진단 결과
- 제품 기본정보 요약
- 이 제품에 필요한 제출 서류 체크리스트 (각 항목에 설명 포함)
- 포장지/물품사양서에서 발견된 주의사항 (규정 위반 가능성, 확인 필요 원료 등)
- 생산자에게 요청할 사항 목록

---

### 기능 B: 서류 검증 — "받은 서류, 다 맞아?"

**사용 시나리오**: 생산자/협력업체로부터 물품사양서 + 증빙서류를 받았을 때, 전국 생소실회의에 올리기 전에 서류가 완비되었는지, 내용이 맞는지 점검

**입력** (모두 업로드):
- 포장지 표시사항 사진 (JPG/PNG)
- 물품사양서 PDF
- 증빙서류 PDF/JPG (복수)

**출력**: 검증 결과
- 포장지 ↔ 물품사양서 내용 일치 여부 대조표
- 물품사양서 기재 완전성 점검 (빠진 항목, 오류)
- 증빙서류 완비 여부 체크리스트 (제출/미제출)
- 증빙서류 적합성 점검 (제품명 일치, 유효기간 등)
- 출하 승인 과정에서 이슈될 수 있는 사항 (미리 대비용)

### 아키텍처 변경점 (기존 대비)

| 항목 | 기존 설계 | 새 설계 |
|---|---|---|
| 대상 사용자 | 전국본부 실무자(토니) | 교구 실무자 (신입 포함) |
| 기능 수 | 2개 (심의 준비 + 심의 판정) | 2개 (빠른 진단 + 서류 검증) |

| 설명 수준 | 전문가 대상 | 신입직원도 이해 가능한 설명 |
| 생산자 요청사항 | 없음 | 복사 가능한 요청 목록 자동 생성 |

### 핵심 변경: engine/checklist_engine.py

기존 `rule_checker.py`의 판정 로직(승인/서류보완/재검토/반려)을 삭제하고,
**체크리스트 항목 + 설명 텍스트** 를 생성하는 엔진으로 교체.

```python
class ChecklistItem:
    category: str          # "필요서류", "기재점검", "원재료주의", "이슈대비"
    status: str            # "required", "submitted", "missing", "warning", "ok"
    title: str             # 항목 제목 (예: "사업자등록증")
    description: str       # 신입직원용 설명 (예: "제조원의 사업자등록증 사본이 필요합니다")
    action: str            # 실무자가 할 행동 (예: "제조원에 사본 요청")
    regulation_ref: str    # 규정 근거 (예: "생산규정 5-2조 6항")
    icon: str              # ✅, ⚠, ❌, 💡

class ChecklistResult:
    product_info: dict                    # 제품 기본정보
    required_documents: List[ChecklistItem]  # 필요 서류 체크리스트
    spec_check: List[ChecklistItem]       # 물품사양서 기재 점검
    ingredient_warnings: List[ChecklistItem] # 원재료 주의사항
    potential_issues: List[ChecklistItem]  # 출하 승인 시 예상 이슈
    request_to_producer: List[str]        # 생산자에게 요청할 사항 (텍스트 목록)
```

### 핵심 변경: 설명 텍스트 데이터

**제출서류_매핑.xlsx에 설명 열 추가**:

| 서류명 | 설명_짧은 | 설명_긴 | 요청방법 |
|---|---|---|---|
| 사업자등록증 | 제조원의 사업자등록증 사본 | 식품을 만드는 회사(제조원)의 사업자등록증 사본이 필요합니다. 세무서에서 발급받은 것으로, 회사명과 대표자명이 물품사양서와 일치해야 합니다. | 제조원에 "사업자등록증 사본"을 요청하세요 |
| Non-GMO 확인서 | 옥수수/대두 원료의 비유전자변형 확인 | 옥수수나 콩(대두)은 유전자변형(GMO) 작물로 재배되는 경우가 많아요. 우리농은 GMO 원료 사용을 금지하므로, Non-GMO(비유전자변형) 확인서가 필요합니다. | 제조원에 "해당 원료의 Non-GMO 확인서 또는 IP(Identity Preserved) 인증서"를 요청하세요 |
